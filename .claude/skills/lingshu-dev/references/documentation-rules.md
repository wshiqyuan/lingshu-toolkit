# Documentation Rules

This reference contains detailed documentation guidelines for the lingshu-toolkit.

## Critical Rule

**The script-generated content in `index.mdx` MUST NOT be modified.** All additional documentation should be appended to the END of the file.

The script generates this structure automatically:

```mdx

## Documentation Sections

### API Section

Document all parameters and return values:

```mdx
## API

| Parameter | Type | Description |
|-----------|------|-------------|
| data | `T` | The input data to process |
| options | `Options` | Optional configuration object |
| options.strict | `boolean` | Enable strict mode (default: false) |

| Return | Type | Description |
|--------|------|-------------|
| result | `Result<T>` | The processed result |
```

**Rules:**
- Use TypeScript type notation
- Document all parameters
- Document return values
- Include default values for optional parameters
- Use clear, concise descriptions

### Examples Section

Provide practical examples showing real-world usage:

```mdx
## Examples

### Basic Usage

```ts
import { toolName } from '@cmtlyt/lingshu-toolkit/shared';

const result = toolName('input data');
console.log(result);
\```

### Advanced Usage

```ts
import { toolName } from '@cmtlyt/lingshu-toolkit/shared';

const result = toolName('input data', {
  strict: true,
  customOption: 'value'
});
console.log(result);
\```

### With React Hook
```tsx
import { useToolName } from '@cmtlyt/lingshu-toolkit/react';

function Component() {
  const [state, actions] = useToolName('default');
  
  return (
    <div>
      <p>State: {state}</p>
      <button onClick={() => actions.set('new value')}>
        Update
      </button>
    </div>
  );
}
\```
```

**Rules:**
- Start with basic usage
- Include advanced usage with options
- Show integration with other tools if applicable
- Use realistic, copy-pasteable code
- Add brief explanations for complex examples

## Common Documentation Patterns

### Utility Function Pattern

```mdx
## API

| Parameter | Type | Description |
|-----------|------|-------------|
| input | `string` | The input string to process |
| options | `ProcessOptions` | Optional processing options |

| Return | Type | Description |
|--------|------|-------------|
| result | `string` | The processed string |

## Examples

### Basic Usage

```ts
import { processString } from '@cmtlyt/lingshu-toolkit/shared';

const result = processString('hello world');
// Returns: "HELLO WORLD"
\```

### With Options

```ts
import { processString } from '@cmtlyt/lingshu-toolkit/shared';

const result = processString('hello world', {
  uppercase: false,
  trim: true
});
// Returns: "hello world"
\```
```

### React Hook Pattern

```mdx
## API

| Parameter | Type | Description |
|-----------|------|-------------|
| defaultValue | `T` | The initial value |

| Return | Type | Description |
|--------|------|-------------|
| state | `T` | Current state value |
| actions | `Actions<T>` | Actions to manipulate state |

### Actions

| Action | Type | Description |
|--------|------|-------------|
| set | `(value: T) => void` | Set state to new value |
| reset | `() => void` | Reset to initial value |
| toggle | `() => void` | Toggle between two values |

## Examples

### Basic Usage

```tsx
import { useBoolean } from '@cmtlyt/lingshu-toolkit/react';

function ToggleButton() {
  const [isOn, { toggle }] = useBoolean(false);
  
  return (
    <button onClick={toggle}>
      {isOn ? 'ON' : 'OFF'}
    </button>
  );
}
\```
```

### Vue Hook Pattern

```mdx
## API

| Parameter | Type | Description |
|-----------|------|-------------|
| defaultValue | `T` | The initial value |

| Return | Type | Description |
|--------|------|-------------|
| state | `Ref<T>` | Reactive state reference |
| set | `(value: T) => void` | Set state to new value |
| reset | `() => void` | Reset to initial value |
| toggle | `() => void` | Toggle between two values |

## Examples

### Basic Usage

```vue
<script setup>
import { useBoolean } from '@cmtlyt/lingshu-toolkit/vue';

const { state, toggle } = useBoolean(false);
</script>

<template>
  <button @click="toggle">
    {{ state ? 'ON' : 'OFF' }}
  </button>
</template>
\```
```
